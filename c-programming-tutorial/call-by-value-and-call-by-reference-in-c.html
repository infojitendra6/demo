Call by value and Call by reference in C

Call by value and Call by reference in C:- In computer programming language there are two ways that arguments can be passed to a subroutine.
There are following two way to pass arguments to a subroutine.
1.Call by value
2.Call by reference

1.Call by value in C programming
In this method, the value of actual parameters is copied into the formal parameters in the function. We can not modify the value of actual parameter by changing the value of formal parameter in program.

In call by value, different memory is allocated by actual and formal parameters, therefore the value of the actual parameter is copied into the formal parameter.

The actual parameter is the argument which is used in function call, whereas formal parameter is the argument which is used in function definition in program.

Call by value example

#include <stdio.h>
void swap(int x, int y); 
int main() 
{ 
 int a, b; 
 printf("Enter two number :");
 scanf("%d%d",&a, &b);
 swap(a, b);  // pass by value
 printf("a=%d b=%d\n", a, b); 
return 0; 
} 
void swap(int x, int y) 
{ 
 int temp; 
 temp = x; 
 x = y; 
 y = temp; 
 printf("\nx=%d y=%d\n", x, y);  
}

Output of program after compilation and execution is given below.

when input is 20 25 
Enter two number : 20 25
x=25 y=20
a=20 b=25
when input is 60 75
Enter two number : 60 75
x=75 y=60
a=60 b=75

Explanation of above program
1.The actual values of a and b remain unchanged , after exchanging the values of x and y in program.
2.In call by values method we cannot alter the values of the actual variables through function calls.
3. The values of variables are passes by Simple technique.

Call by reference in C programming
In Call by reference method, the address of the variable is passed into function call as the actual parameter.

The value of the actual parameters of function can be modified by changing the value of the formal parameters in program since the address of the actual parameters is passed.

In call by reference, Memory allocation is similar for both formal parameters and actual parameters.

Call by reference example

#include <stdio.h>
void swap(int*x, int*y); 
int main() 
{ 
 int a, b; 
 printf("Enter two number :");
 scanf("%d%d",&a, &b);
 swap(&a, &b); // Pass reference
 printf("a=%d b=%d\n", a, b); 
	return 0; 
} 
void swap(int* x, int* y) 
{ 
 int temp; 
 temp = *x; 
 *x = *y; 
 *y = temp; 
 printf("\nx=%d y=%d\n", *x, *y); 
}

Output of program after compilation and execution is given below.

when input is 20 25 
Enter two number : 20 25
x=25 y=20
a=25 b=20
when input is 60 75
Enter two number : 60 75
x=75 y=60
a=75 b=60

Explanation of above program
The actual values of a and b get changed after exchanging values of x and y in program, because the formal parameters and the actual parameters use same memory location.

There are following difference between call by value and call by reference in C

Call by value Call by reference
In this method, a copy of the value is passed into the functions. In this method, an address of value is passed into the functions.
In this method, the values of actual parameters do not change by changing the formal parameters in program. In this method, the values of actual parameters do change by changing the formal parameters.
The actual and formal arguments are created at the different memory location in ROM. The actual and formal arguments are created at the same memory locationin ROM.
